# 安全漏洞报告

**请不要用公开 issue 报告安全漏洞**——包括凭据泄露、越权、可被利用的缺陷。公开 issue 会让漏洞在修复前暴露。

## 怎么报告

1. **已开启「私有漏洞报告」的仓**（该功能需在组织 / 仓级显式启用，public 仓也不例外）：在该仓 **Security** 标签页 →「**Report a vulnerability**」（GitHub 私有报告，仅维护者可见）。
2. **其他所有情况**（看不到上述入口——无论 public 还是 private 仓）：通过组织维护者提供的私有安全渠道上报——**该渠道（安全团队邮箱 / 内部工单）由 switchbot-rd 维护者在落地时填入此处**；补全前请直接私信你所在仓的维护者转交。**任何情况下都不要开公开 issue。**

## 报告时请附上

- 受影响的仓库 / 组件，以及版本或 commit。
- 复现步骤或 PoC——**务必脱敏**，不要包含真实的凭据、密钥、内网地址等敏感信息。
- 你评估的影响范围与严重程度。

我们会尽快确认并跟进修复。**在修复发布前，请勿公开披露。**
