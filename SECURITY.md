# 安全漏洞报告

**安全问题（凭据泄露、越权、可被利用的缺陷）请走私有渠道，不要开普通 issue。** 即便仓是私有的，普通 issue 也对所有能访问该仓的人可见——不该让漏洞细节躺在那里。

## 怎么报告

通过组织维护者提供的私有安全渠道上报——**该渠道（安全团队邮箱 / 内部工单）由 switchbot-rd 维护者在落地时填入此处**；补全前请通过内部渠道私下联系你所在仓的维护者转交。

> 少数 public 仓若已开启 GitHub「私有漏洞报告」，也可在该仓 **Security** 标签页 →「Report a vulnerability」直接上报（该入口仅 public 且显式启用的仓才有）。

## 报告时请附上

- 受影响的仓库 / 组件，以及版本或 commit。
- 复现步骤或 PoC——**务必脱敏**，不要包含真实的凭据、密钥、内网地址等敏感信息。
- 你评估的影响范围与严重程度。

我们会尽快确认并跟进修复。**在修复发布前，请勿公开披露。**
